Usage

How to use the skills

The skills are conversational — no forms to fill out. Describe what you need and Claude activates the right skill automatically.

1

Install the skills

These are shared skills on Claude Enterprise — your org admin installs them once and they're available to everyone on the team automatically. No individual downloads needed. If you don't see them in Claude, ask your Claude Enterprise admin to install the .skill files via the Admin Console → Skills.

2

Run product-marketing-context first

Open Claude and ask it to set up your product marketing context. It will walk through a structured set of questions — product, ICP, differentiators, competitors, GTM motion — and save everything to a context file. Do this once; all other skills read from it automatically.

"Set up product marketing context for Applause's agentic AI testing solution"
3

Trigger any skill with plain language

Start a Claude conversation and describe your task. The skill recognizes the request and activates automatically — no slash commands required. Be specific about your goal, target audience, or deal context for sharper output.

"Build a messaging framework for our AI testing solution targeting QA Directors"
"Create a GTM plan for launching Payment Testing to financial services"
"Build buyer personas for engineering leaders evaluating crowdtesting"
4

Review, refine, and export

All document and visual outputs follow Applause brand guidelines — colors, typography, and voice — automatically. Download Word docs, PDFs, or copy structured content directly into your deliverables.

Best Practices

Getting the most out of it

A few patterns that make all the difference.

🗺️
Context file = force multiplier
The richer your product-marketing-context file, the less every other skill needs to ask. Invest 20 minutes in it upfront — it pays off across every GTM, messaging, and research task you run.
🎯
Add solution area context
Specifying the solution area ("for a payments conversation" or "for an agentic AI deal") gives you sharper, more targeted output than a generic request. Applause's breadth is a strength — use it.
🔄
Chain skills together
Run customer-research → messaging-positioning → go-to-market in sequence. Each skill builds on the last, and the shared context file keeps everything consistent. This is the full PMM workflow in Claude.
⚡
Pair with Battlecard Builder
competitive-intelligence feeds directly into the Battlecard Builder ↗. Use the CI skill for research and analysis; use Battlecard Builder for formatted .docx output ready for reps.
✏️
Outputs are starting points
Treat skill output as a strong first draft, not a final. Human review — especially for win/loss stories and unverified competitor claims — is always the last step before field use.
🔁
Update context quarterly
Re-run product-marketing-context when positioning shifts, you enter new markets, or launch major features. Stale context degrades all downstream skill output.
