The Program

How the three skills connect

The skills are designed to run in sequence. Each one feeds the next.

Run the LLM visibility analyzer first — it tells you which topics and categories Applause is invisible in and why. Use those findings to prioritize which pages need attention.

Then run the page-level audit on each priority page. The visibility report sets the strategy; the audit builds the specific implementation brief for each page.

Before anything goes to a stakeholder, run the evaluator. It catches outdated guidance, flags unsupported claims, and gives you a distribution verdict.

Strategy sets the direction. The audit builds the brief. The evaluator gates distribution.

LLM Visibility Analyzer Answers: where are we invisible and why?
Page-level Audit Answers: how do we fix this specific page?
Usage

How to run the program

The skills are conversational — no forms, no menus. Add them from the Claude Enterprise Skills Library and describe your task.

1

Add the skills from the Claude Enterprise library

In Claude Enterprise, open the Skills Library and search for each skill name. Click Add to Project for your PMM Project. If a skill isn't visible, ask your Claude Enterprise admin to publish it from the Admin Console.

2

Run the LLM visibility analyzer first

Export the citation tracker as CSV and upload it to Claude. This is your strategic starting point — it tells you which topics are complete blackouts and which competitors are dominating.

"Analyze this citation tracker CSV and produce an LLM visibility report for Applause.
Focus on blackout categories and the highest-priority content gaps."
3

Use the visibility report to prioritize page audits

Pick the pages that map to your highest-priority gap topics. Paste the URL into Claude and run the page-level audit. The skill fetches the page content and audits it automatically.

"Audit applause.com/payment-testing/ for AI search visibility.
Focus on the 'payment testing' and 'mobile payments QA' queries."
4

Run the evaluator before distributing

Paste the page audit report into Claude and run the evaluator. It scores the report across six dimensions and returns a distribution verdict. Only share reports that pass.

"Run the AI SEO AEO evaluator on this page audit report.
Flag anything that needs revision before I share it with the web team."
5

Track, implement, and rerun the pulse check

Hand off the audit action plan to the web team with P1/P2/P3 owners. Run the monthly pulse check (9 queries × 3 platforms) to track citation rate movement against the June 2026 baseline.
